- **Focus mechanism (the load-bearing new capability):** `scripts/test.mjs`
  turns any words after `npm test --` into a Playwright `--grep`, and writes a
  deterministic **run-record** (`.trellis/last-run.json`) naming exactly which
  tests the LAST run executed and whether they passed. This makes focused-run
  *identity* measurable — the session event log only carries aggregate
  `{passed,failed}` counts and cannot prove which single test ran.

  (one test, not the suite → the scenario's `wrong-scope-accepted` blocker),
  the named test, the pass, and byte-for-byte integrity of the tests + page
  (editing the suite to leave one runnable test → the `project-mutated`
  blocker). Session `ran-tests` + `repo-valid` round out the checkpoint.
- Catalog entry added to `apps/web/src/scenarios.ts` (QA & Testing /
  Playwright / beginner).

Pre-flight validated end-to-end through the public API against a live docker
container: full-suite run → checkpoint rejects wrong scope; focused run → all
